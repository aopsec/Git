# OpenBox Roadmap

## Phase 1
- Stabilize bootstrap behavior.
- Align docs, configs, and runtime ordering.
- Replace placeholder validation with deterministic checks.

## Phase 2
- Capture performance baselines for WireGuard + CAKE.
- Formalize security decisions with ADRs.
- Tighten CI around systemd, Tor, Caddy, and the generated vault.

## Phase 3
- Add hardware validation results from a real Raspberry Pi target.
- Track repeatable benchmarks and incident notes in the vault.
- Promote validated example configs into safer install defaults.
